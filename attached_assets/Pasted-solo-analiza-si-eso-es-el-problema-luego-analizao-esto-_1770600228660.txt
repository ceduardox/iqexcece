1. ¿Cómo modifica la Base de Datos?
Bolt.diy (y los agentes avanzados) no entran a tu base de datos con un mouse. Lo hacen a través de código y terminal:

Si usas Migraciones (Prisma, TypeORM, Knex): Es donde mejor funciona. La IA modifica el archivo de esquema (ej. schema.prisma), y luego ella misma ejecuta en la terminal integrada: npx prisma migrate dev.

Si usas SQL puro: La IA buscará tu archivo de conexión o tus scripts de inicialización, añadirá la columna o tabla nueva, y ejecutará el script usando comandos de terminal (ej. psql o node seed.js).

Postgres en Replit: Como sabe que estás en un entorno Node, usará las credenciales de tu .env para conectarse y ejecutar las queries necesarias para actualizar la estructura.

2. ¿Cómo reinicia el Servidor?
Bolt.diy tiene control sobre la Terminal, por lo tanto:

Auto-Restart: Si tu proyecto usa nodemon o tsx --watch, el servidor se reiniciará solo en cuanto la IA guarde los cambios en el código.

Reinicio Manual: Si el servidor se queda colgado por un error que ella misma cometió, la IA puede detectar que el proceso murió, hacer un Ctrl+C virtual y ejecutar npm run dev de nuevo.

Detección de Crash: Si al reiniciar el servidor sale un error de "Puerto ya en uso" o "Error de sintaxis", la IA lee ese log de error y no se queda esperando; intenta corregir el código inmediatamente para que el servidor vuelva a levantar.

3. El gran "PERO" (Lo que debes vigilar)
Aunque Bolt.diy sea muy listo, tiene un riesgo con las Bases de Datos:

Peligro de "Data Loss": Si la IA decide que para "arreglar" un problema debe borrar una tabla y crearla de nuevo, lo hará. La solución: Debes decirle en sus reglas (System Prompt): "NUNCA borres tablas con datos reales, siempre usa comandos ALTER para modificar la estructura".
